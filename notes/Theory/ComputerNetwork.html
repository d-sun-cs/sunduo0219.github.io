                                                    # Computer Network

                                                    # Overview

                                                    # Concept & Classification

                                                    # Concept:

                                                    • Concise concept: a set of interconnected and autonomous computers

                                                      • Interconnected: communication link
                                                      • Autonomous: no master-slave relation

                                                    • Complete concept: a system connecting distributed and independent computer systems by communicative devices and links, and implement resource sharing and information transfer by software

                                                      • Computer systems / Terminal systems: mobile, PC, ...
                                                      • Communicative devices: switch, router, ...
                                                      • Communicative links: mediums connecting terminal systems and communicative devices
                                                      • Topological structure & Software: The topological formed by terminal systems, communicative devices and links can implement functions only by software

                                                    # Classification:

                                                    • Classified by distribution range

                                                      • Wide area network (WAN)
                                                        • Range: tens to thousands of kilometers, large, multinational
                                                        • Prime technology: switch
                                                        • Status: the core of internet, high speed links, large communication capacity
                                                      • Metropolitan area network (MAN)
                                                        • Range: five to fifty kilometers
                                                        • Prime technology: ethernet
                                                        • Status: always merged into LAN
                                                      • Local area network (LAN)
                                                        • Range: tens to thousands of meters
                                                        • Prime technology: broadcast
                                                      • Personal area network
                                                        • Range: ten meters
                                                        • Prime technology: wireless

                                                    • Classified by user

                                                      • Public network: large network constructed by telecom companies
                                                      • Private network: constructed for business requirements of specific unit

                                                    • Classified by topological structure

                                                      • Bus network
                                                      • Star network: nnn nodes, n−1n-1n−1 links
                                                      • Ring network
                                                      • Net network: always used in WAN like internet

                                                    • Classified by switch technology

                                                      • Circuit switching network
                                                      • Message switching network
                                                      • Packet switching network

                                                      More detailed introductions later ...

                                                    • Classified by transfer technology

                                                      • Broadcast network: communication channel shared by all hosts

                                                      • Point to point network: each link connecting a pair of computers

                                                        packet store-and-forward and routing

                                                    • How’s computer network composed:

                                                      • Divided by facility

                                                        • Hardware:

                                                          • Host / Computer system / Terminal system
                                                          • Communicative devices / Switching devices
                                                          • Communicative links

                                                        • Software:

                                                          • For resource sharing

                                                          • Tools for users

                                                            e.g. email, ftp, chat, ...

                                                        • Protocol: the core of computer network

                                                      • Divided by working

                                                        • Edge: hosts

                                                        • Core: networks and routers

                                                          Provide connectivity and switch services for edge parts

                                                      • Divided by function

                                                        • Communication subnet: data communication, transmission media, communication devices, protocols
                                                        • Resource subnet: resource sharing and data processing, devices and software

                                                        The layered models are just divided by function

                                                    # Standardization & Layered Model

                                                    # Standardization:

                                                    • Why to standardize: follow the uniform standards to realize the interconnection between different hardware and software from different manufacturers, promoting the development of computer network

                                                    • What category of standards:

                                                      • Mandatory standards: official and legal standards set by authorities

                                                        For example, OSI/ISO standard

                                                      • DE facto standards: standards in some products of companies dominating the competition

                                                        For example, TCP/IP standard

                                                    • How to set standards:

                                                      • RFC (Request For Comments)

                                                        1. Internet draft
                                                        2. Proposed standard: RFC document
                                                        3. (Draft standard)
                                                        4. Internet standard: audited by IETF and IAB

                                                      • Related organizations:

                                                        • ISO: International Standardization Organization

                                                          OSI: Open System Interconnection

                                                          HDLC: High-level Data Link Control

                                                        • ITU: International Telecommunication Union

                                                        • IEEE: Institute of Electrical and Electronics Engineers

                                                        • IETF: the Internet Engineering Task Force

                                                        • IAB: Internet Architecture Board

                                                    # Layered Model:

                                                    • Layering:

                                                      • What is / Why to layer: an abstract computer network architecture describing functions of network, to divide and conquer

                                                      • What’s inside layers:

                                                        • Entity: PDU(Protocol Data Unit) = SDU(Service Data Unit) + PCI(Protocol Control Information)

                                                          • PDU’s “nickname” in different layers:
                                                            • “bit” in physical layer
                                                            • “frame” in link layer
                                                            • “packet” in network layer
                                                            • “message” in transport layer

                                                        • Network protocol: horizontal rules or appointments to exchange data between peer entities

                                                          • Grammar
                                                          • Semantics
                                                          • Sync

                                                        • Interface: SAP(Service Access Point) where vertical entities exchange data, or the upper layer calls the lower layer

                                                        • Service: “vertical” function calling served by the lower layer for the upper layer

                                                      • How to layer:

                                                        • Principles:

                                                          • Independence and complete, with appropriate technologies
                                                          • Clear boundaries and interfaces
                                                          • Standardization

                                                        • Models: ISO/OSI, TCPIP

                                                    • ISO/OSI:

                                                      • Where’s ISO/OSI from: ISO raised it to support the interconnection of heterogeneous network systems
                                                      • How does ISO/OSI layer:

                                                        • Communication subnet:

                                                          • Physical layer: transparent bit transfer on physical mediums

                                                            • Interface feature
                                                            • Transmission mode
                                                            • Bit encoding and syncing

                                                          • Data link layer: framing

                                                            • Error control
                                                            • Flow control
                                                            • Media access control

                                                          • Network layer: transmit packets from source to destination according to protocols like IP

                                                            • Routing
                                                            • Error control
                                                            • Flow control, congestion control

                                                        • Transport layer: peer-to-peer communication, between processes, with the unit of message according to protocols like TCP and UDP

                                                          • Reliable or reliable transmission
                                                          • Reuse and multiplexing
                                                          • Error control
                                                          • Flow control

                                                        • Resource subnet:

                                                          • Session layer: connection establishment
                                                            • Checkpoint
                                                          • Presentation layer: Handle information representing between two communication systems
                                                            • Data format transformation
                                                            • Data encryption and decryption
                                                            • Data compression and decompression
                                                          • Application layer: communication between application programs according to protocols like FTP, SMTP, and WWW

                                                    • TCP/IP:

                                                      • Where’s TCP/IP from: TCP/IP protocol stack
                                                      • How does TCP/IP layer:

                                                        • Network interface layer: physical and data link layer

                                                          • Protocols: Ethernet, ATM, Frame Relay

                                                        • Internet layer: network layer

                                                          • Protocols: IP

                                                        • Transmission layer: transport layer

                                                          • Protocols: TCP, UDP

                                                        • Application layer: session, presentation and application layer

                                                          • Protocols: HTTP, FTP, DNS

                                                    ISO/OSI vs TCP/IP

                                                    • Similarity:
                                                      • Layer and protocol stack
                                                      • Interconnection of heterogeneous network
                                                    • Difference:
                                                      • ISO/OSI: model over protocol
                                                      • TCP/IP: value IP for interconnection of heterogeneous network initially
                                                      • Different communication ways in network and transport layer: connection-oriented or connectionless
                                                        • connection: establish, transmit and release

                                                        • Baseband signal: Use single frequency and transmit binary data directly by different voltage on digital channel of both way

                                                        • Broadband signal: Use analog signal of multiple frequencies to transmit data on analog channel of single way

                                                    # Encoding & Modulation:

                                                    • What’s encoding & modulation:

                                                      • Encode: Transform analog or digital data into digital signal
                                                      • Modulate: Transform analog or digital data into analog signal

                                                    • How to encode:

                                                      • Encode digital data into digital signal: Use digital transmitter

                                                        • NRZ(Non Return to Zero): Use two levels to represent binary directly

                                                          • Disadvantages: Unable to detect error. Difficult to sync

                                                        • RZ(Return to Zero): Return to zero in each symbol based on NRZ

                                                          • Advantages: Able to Sync
                                                          • Disadvantages: Always zero, low transmission efficiency

                                                        • NRZI(Non Return to Zero-Inverted): Use level reversal or not to represent binary

                                                          • Advantages: Able to Sync. No bandwidth loss

                                                          USB 2.0

                                                        • Manchester Encoding: Jump in the middle of a symbol and use high to low or low to high to represent binary

                                                          • Advantages: Self-sync
                                                          • Disadvantages: Double bandwidth

                                                          Ethernet

                                                        • Differential Manchester Encoding: Use level reversal or not to represent binary based on Manchester Encoding

                                                          • Advantages: Anti-interference

                                                        • 4B/5B Encoding: Use 5 bits to represent 4 bits data

                                                          • Advantages: Other 16 type control code

                                                      • Encode analog data into digital signal: Use PCM(Pulse Code Modulation) encoder

                                                        1. Sample: Transform continuous signal in time into discrete signal in time by scanning analog signal periodically
                                                          • Sample Theorem / Nyquist Theorem: fsample≥2fbandwidthf_{sample}\ge2f_{bandwidth}fsample​≥2fbandwidth​ to keep lossless
                                                        2. Quantify: Transform level amplitude collected by sampling into number
                                                        3. Encode: Transform quantified result into binary encoding

                                                    • How to modulate:

                                                      • Modulate digital data into analog signal: Use modulator
                                                        • ASK(Amplitude Shift Keying)
                                                          • Advantages: Simple to implement
                                                          • Disadvantages: Poor anti-interference
                                                        • FSK(Frequency Shift Keying)
                                                          • Advantages: Simple to implement. Good anti-intererence
                                                        • PSK(Phase Shift Keying):
                                                          • Types: Absolute & Relative
                                                        • QAM(Quadrature Amplitude Modulation): ASK + PSK
                                                          • R=Blog2(mn)R=Blog_2(mn)R=Blog2​(mn)
                                                      • Modulate analog data into analog signal: Use amplifier modulator and FDM(Frequency Division Multiplexing)

                                                    # Switching Technology:

                                                    • What’s switching: Data transmission between nodes in communication subnet
                                                    • How to switch:
                                                      • Circuit switching: Apply to large data volume
                                                      • Message switching: Apply to multiple links and devices
                                                      • Packet switching: Apply to multiple links and devices and bursty communication
                                                        • Datagram
                                                        • Virtual circuit
                                                    • What’s circuit switching: Construct a specified and exclusive data communication path crossing multiple switching devices and links
                                                      • Advantages:
                                                        • Easy
                                                        • Low delay
                                                        • Ordered
                                                        • No conflict
                                                        • Applied to both digital and analog signal
                                                      • Disadvantages:
                                                        • Construction time
                                                        • Low utilization rate for exclusion
                                                        • Low flexibility
                                                        • Difficult to do error control
                                                    • How to do circuit switching:
                                                      1. Construct connection
                                                      2. Transfer data
                                                        • Occupy fixed end-to-end bandwidth
                                                        • No store and forward
                                                      3. Release connection
                                                    • What’s message switching: Store and forward in messages carrying target address and source address by network switching nodes
                                                      • Advantages:
                                                        • Not construct connection
                                                        • Dynamic link allocation by switching devices
                                                        • High reliability for dynamic link
                                                        • High utilization rate for not exclusion
                                                        • Multi-target service
                                                      • Disadvantages:
                                                        • Delay for storing and forwarding
                                                        • Cache for packets
                                                    • What’s packet switching: Store and forward in packets with limited size and control information by network switching nodes
                                                      • Advantages:
                                                        • Not construct connection
                                                        • High utilization rate for not exclusion
                                                        • Simplify storage management for cache
                                                        • Accelerate transmission for parallel packets
                                                        • Low error and retry for smaller packet
                                                      • Disadvantages:
                                                        • Delay for storing and forwarding
                                                        • Extra information
                                                        • Disordered packets
                                                        • Also construct and release connection for virtual circuit
                                                    • How to do packet switching:
                                                      • Datagram: Connectionless packet transmission
                                                        • Need to do error detecting and routing
                                                        • Not ensure order
                                                      • Virtual circuit: A logical circuit path from source host to target host.
                                                        1. Construct virtual circuit: Send call request packet and receive call response packet
                                                          • All nodes on the path maintain a virtual circuit table and make virtual circuit number when connection is being constructed
                                                          • One node can participate multiple virtual circuits
                                                        2. Transfer data: Transfer ordered packets
                                                          • Not carry addresses and route , just use virtual circuit number
                                                          • Need to detect error
                                                          • Sigle point of failure
                                                        3. Release virtual circuit

                                                    # Framing

                                                    • What’s framing: Combine bits into frames as unit for transmission by add a head and a tail for a piece of data

                                                    • Why to frame:

                                                      • Frame delimitation: Only retransmit a frame but not total data when frame error happens
                                                      • Frame synchronization: Receiver can distinguish start and end of frame from binary bit flow
                                                      • Transparent transmission: Data link can transmit any bit combination

                                                    • How to frame:

                                                      • Character count: Add a count field in frame head to mark the number of characters

                                                        • Disadvantages: Error happens in count field

                                                      • Flag characters with character stuffing: Use specified character STX and ETX to mark head and tail, and append escape character ESC to mark specified character itself

                                                      • Zero bit stuffing: Use 01111110 to mark head and tail and append a 0 per 5 1

                                                        Easy to implement by hardware

                                                      • Physical layer coding violations: Use illegal level pair of physical level

                                                        • Only apply to redundancy encoding

                                                        LAN IEEE 802 adopts it

                                                    # Error Control

                                                    • What’re types of noise:

                                                      • Random noise: Generated from electrical characteristics of channel itself
                                                        • Solved by improving sensor to enhance SNR
                                                      • Impulse noise: Generated from outside specified factor. The main cause of error
                                                        • Solved by encoding

                                                    • What’re types of error:

                                                      • Bit error
                                                      • Frame error:
                                                        • Loss
                                                        • Repeat
                                                        • Disorder

                                                    • How to solve error: Redundancy encoding

                                                      • Error detection code:
                                                        • Parity check code
                                                        • CRC(Cyclic Redundancy Code)
                                                      • Error correction code:
                                                        • Hamming code

                                                    • What’s parity check code: n−1n-1n−1 information bits and 111 check bit

                                                      • Only able to detect error of odd number of bits

                                                        5050%50 capability of error detection

                                                      • Unable to correct the error bit

                                                    • What’s CRC: Given mmm bit frame, sender generates rrr bit sequence called as FCS(Frame Check Sequence) to form a m+rm+rm+r bit frame which is divisible by G(x)G(x)G(x) agreed by sender and receiver

                                                      • Detect error of odd number, double bit and continuous error less than or equal to the length of parity bit
                                                      • Correct one bit error if 2r≥m+r+12^r\ge m+r+12r≥m+r+1

                                                    • How to use CRC:

                                                      • Calculate FCS: Given an mmm bit frame and the corresponding polynomial M(x)M(x)M(x)

                                                        1. Append 0: Append rrr 0s to the low bit if the rank of G(x)G(x)G(x) is rrr to get M′(x)M'(x)M′(x)
                                                        2. Divide modulo 2: FCS=M′(x)%G(x)FCS=M'(x)\%G(x)FCS=M′(x)%G(x)
                                                          • Copy the highest bit
                                                          • Exclusive or

                                                      • Error detection: Check if M(x)FCS %G (x)==0M(x)FCS~\%G~(x)==0M(x)FCS %G (x)==0 (modulo 2)

                                                        • If true, accept the frame

                                                        • if false, discard and retransmit the frame

                                                          Or correct error with remainder

                                                          The relation between remainder and error bit is confirmed by G(x)G(x)G(x)

                                                      Hardware can generate FCS and do checking

                                                    • What’s hamming code: Divide nnn bits into kkk groups and set a parity bit for each of kkk groups

                                                      • Detect two error bits with full parity bit
                                                      • Correct one error bit

                                                    • How to use hamming code:

                                                      • Calculate hamming code:
                                                        1. Amount of parity bits: 2k≥n+k+12^k\ge n+k+12k≥n+k+1
                                                        2. Distribution of parity bits: Put PiP_iPi​ in position 2i−12^{i-1}2i−1
                                                        3. Value of parity bits: PiP_iPi​ is equal to exclusive or of information bit(DjD_jDj​) taking 111 in ithithith position (even parity check)
                                                        4. (Append a full parity bit)
                                                      • Error correction:
                                                        1. Exclusive or PiP_iPi​ and related DjD_jDj​ to get SiS_iSi​
                                                        2. Rank SiS_iSi​ from high bit to low bit to get the error bit
                                                        3. (Check full parity bit)
                                                          • If one bit error, correct it
                                                          • If double bit error, discard and retransmit frame

                                                    # Flow Control & Reliable Data Transfer

                                                    • What’s flow control & reliable data transfer:

                                                      • Limit data flow of sender not to over the capability of receiver
                                                      • Receiver receives the same data as sender sends

                                                    • Why to do flow control & reliable data transfer:

                                                      • Error control can’t correct all bit errors, retransmission is still required
                                                      • There will be packet loss:
                                                        • Error will happen if sending speed doesn’t match receiving capability
                                                        • Also caused by physical link failure, device failure, virus attack, wrong route and so on

                                                    • How to do flow control and reliable data transfer: Acknowledgement mechanism, ARQ(Auto Repeat reQuest) and sliding window protocol

                                                      • Acknowledgement mechanism: Receiver responses ACK(i) after receiving Frame(i) from sender
                                                      • Stop-and-Wait Protocol:
                                                        • Sender’s window size: =1
                                                        • Receiver’s window size: =1
                                                      • GBN(Go-Back-N protocol):
                                                        • Sender’s window size: >1
                                                        • Receiver’s window size: =1
                                                      • SR(Selective Repeat protocol):
                                                        • Sender’s window size: >1
                                                        • Receiver’s window size: >1

                                                    • What’s Stop-and-Wait protocol: Sender stops and waits for acknowledgement of receiver after sending each packet

                                                    • How does Stop-and-Wait protocol work:

                                                      • Frame sequence number:

                                                        • One bit is enough, because sender will stop and wait
                                                        • ACK from receiver doesn’t need number

                                                      • Sender:

                                                        • Set a timer for each frame, retransmit if ACK fails by timeout

                                                      • Receiver:

                                                        • Discard wrong frame
                                                        • Discard duplicate frame and retransmit ACK

                                                      • Channel utilization rate:

                                                        U=T0TD+RTT+TA U=\frac{T_0}{T_D+RTT+T_A} U=TD​+RTT+TA​T0​​

                                                    • Why we need GBN:

                                                      • Channel utilization rate of Stop-and-Wait protocol is too low
                                                      • Pipeline sending

                                                    • What’s GBN protocol: Sender can send frames continuously but go back NNN frames to retransmit if the previous one fails

                                                    • How does GBN protocol work:

                                                      • Size:

                                                        • Frame sequence number: From 000 to k−1k-1k−1 circularly

                                                        • Sliding window size: Wr+1≤2kW_r+1\le2^kWr​+1≤2k

                                                          Why not equal? There are two cases which can’t be distinguished:

                                                          • Get all ACKs and transmit a new window of frames
                                                          • Lose all ACK and transmit the whole old window of frames

                                                      • Sender:

                                                        • Sending sliding window:

                                                          • Send: ACK, sent but not ACK, usable but not sent, unusable (in order)
                                                          • Receive ACK and slide: Cumulative ACK

                                                        • Timer: Set a timer for each frame, retransmit from Frame(i) to the end of window if ACK(i) fails by timeout

                                                          Even though some frames after i have been transferred successfully

                                                      • Receiver:

                                                        • Cumulative ACK
                                                        • Expected sequence number: Receive frames sequentially, discard any wrong frame and retransmit ACK(i) for recent received frame

                                                      • Channel utilization rate:

                                                        U=WS×tsegtseg+RTT+tACK U=\frac{W_S\times t_{seg}}{t_{seg}+RTT+t_{ACK}} U=tseg​+RTT+tACK​WS​×tseg​​

                                                    • Why we need SR:

                                                      • In GBN, frames which have been transferred successfully may still be retransmitted

                                                    • What’s SR protocol: Sender can send frames continuously and only repeat wrong frame selectively

                                                    • How does SR protocol work:

                                                      • Size:
                                                        • Frame sequence number: From 000 to k−1k-1k−1 circularly
                                                        • Sliding window: WS+WR≤2kW_S+W_R\le2^kWS​+WR​≤2k
                                                          • Usually, WS=WRW_S=W_RWS​=WR​
                                                      • Sender:
                                                        • Sending sliding window:
                                                          • Send: ACK, sent but not ACK, usable but not sent, unusable (out of order)
                                                          • Receive ACK and slide: Slide if the frame sequence number of ACK(i) is the lower bound of window
                                                        • Timer: Set a timer for each frame, retransmit only Frame(i) if ACK(i) fails by timeout
                                                      • Receiver:
                                                        • Receive sliding window:
                                                          • Receive and response ACK: Cache any right frame (within the window)
                                                          • Slide: Slide after receiving continuous frames beginning from the lower bound
                                                        • Discard:
                                                          • Discard frames below the window and response ACK(i) for them
                                                          • Discard wrong frames and frames over the window

                                                    # Medium Access Control

                                                    • What’s MAC / Why to do MAC: Medium access control, a sublayer of data link layer

                                                      • Isolate signal from other nodes in the same channel for every node

                                                    • How to do MAC:

                                                      • Static channel partition: Fixed allocation of channel
                                                        • Channel partition / Multiplexing
                                                          • FDM
                                                          • TDM
                                                          • WDM
                                                          • CDM
                                                      • Dynamic channel allocation: Unfixed allocation of channel
                                                        • Polling access:
                                                          • Token-passing protocol
                                                        • Random access:
                                                          • ALOHA
                                                          • CSMA
                                                          • CSMA/CD
                                                          • CSMA/CA

                                                    • Why to do channel partition / multiplexing:

                                                      • Allocate time and frequency domain resource reasonably

                                                    • What’s channel partition medium access control / multiplexing:

                                                      • Channel partition: Divide broadcast link into noninterfering logical point-to-point link by time division, frequency division and so on
                                                      • Multiplexing: Combine multiple signals into one physical channel to transfer so that multiple terminals can share channel resources

                                                    • How to do channel partition / multiplexing:

                                                      • FDM(Frequency Division Multiplexing): Occupy different frequency bandwidth resource at the same time

                                                        • Easy, high efficiency for full use of bandwidth

                                                      • TDM(Time Division Multiplexing): Divide time into TDM frames and occupy a time slot with fixed sequence number in turn

                                                        • STDM(Statistical TDM): Allocate time slot on demand dynamically to increase use ratio of channel
                                                        • TDM frame also marks a cycle of bit flow in physical

                                                      • WDM(Wavelength Division Multiplexing): Implement FDM of light by optical multiplexer and de-multiplexer

                                                      • CDM(Code Division Multiplexing): Use different encoding schemes to distinguish a **same signal **at the same time and frequency.

                                                        CDMA(Code Division Multiple Access) is a common way of CDM:

                                                        • Chip: A binary bit sequence. Every terminal owns a chip and sending chip for 1 and ~chip for 0

                                                          In chip, -1 is used to substitute 0

                                                    • What’s random access MAC: All terminals transmit data into channel randomly and occupy total bandwidth when transmitting

                                                    • How to do random access MAC:

                                                      • ALOHA:

                                                        • Pure ALOHA:

                                                          • Sender: Transmit data randomly and retransmit after a random period if no ACK for a while
                                                          • Receiver: If any collision detected, no ACK

                                                          Low throughput and efficiency

                                                          S=Ge−2GS=Ge^{-2G}S=Ge−2G

                                                        • Slotted ALOHA:

                                                          • Sender: Terminal can only access channel at the beginning of time slot. If no ACK, retransmit at the next time slot with probability ppp
                                                          • Receiver: If any collision detected, no ACK

                                                          S=Ge−GS=Ge^{-G}S=Ge−G

                                                      • CSMA(Carrier Sense Multiple Access):

                                                        CS: Listen channel before transmitting frames

                                                        MA: Bus

                                                        • 1-persistent CSMA:

                                                          • Sender: Listen channel, if available, transmit, if not, listen until available. If no ACK, repeat to listen after a random period
                                                          • Receiver: If any collision detected, no ACK

                                                        • Non-persistent CSMA

                                                          • Sender: Listen channel, if available, transmit, if not, listen after waiting for a random period. If no ACK, repeat to listen after a random period
                                                          • Receiver: If any collision detected, no ACK

                                                        • P-persistent CSMA

                                                          • Sender: Listen channel, if available, transmit with probability ppp and transmit at the next time slot with probability 1−p1-p1−p, if not, listen until available. If no ACK, repeat to listen at the next time slot
                                                          • Receiver: If any collision detected, no ACK

                                                      • CSMA/CD(CSMA with Collision Detection):

                                                        CD: listen channel to detect collision while transmitting (half-duplex)

                                                        • Contention period and minimum frame length:

                                                          • 2τ2\tau2τ, τ\tauτ is propagation delay

                                                          • min frame len=τ×rate×2min~frame~len=\tau\times rate\times2min frame len=τ×rate×2

                                                            64B in ethernet

                                                        • Sender: Listen channel, if available, transmit and listen at the same time. If collision detected, retransmit by truncated binary exponential back-off algorithm

                                                        Bus ethernet

                                                      • CSMA/CA(CSMA with Collision Avoidance):

                                                        1. RTS(Request To Send)
                                                        2. CTS(Clear To Send)
                                                        3. Reserve channel
                                                        4. ACK and retransmit by truncated binary exponential back-off algorithm

                                                        Wireless LAN

                                                    • What’s polling access MAC: No collision and full bandwidth by polling

                                                    • How to do polling access MAC:

                                                      • Polling protocol: Prime node takes turns to authorize other nodes to transmit data

                                                        Cost of polling, waiting delay, single point failure

                                                      • Token passing protocol: Pass token in turn to transmit data

                                                        • Token: A special form of MAC control frame
                                                        • Physical star topological structure and logical ring topological structure

                                                        Cost of token, waiting delay, single point failure
