                                    # Computer Network

                                    # Overview

                                    # Conceptual Framework

                                    • Concept:

                                      • Concise concept: a set of interconnected and autonomous computers

                                        • Interconnected: communication link
                                        • Autonomous: no master-slave relation

                                      • Complete concept: a system connecting distributed and independent computer systems by communicative devices and links, and implement the function of resource sharing and information transfer by software

                                        • Computer systems / Terminal systems: mobile, PC, ...
                                        • Communicative devices: switch, router, ...
                                        • Communicative links: mediums connecting terminal systems and communicative devices
                                        • Topological structure & Software: The topological formed by terminal systems, communicative devices and links can implement functions only by software

                                    • Composition:

                                      • Divided by component

                                        • Hardware:

                                          • Host / Computer system / Terminal system
                                          • Communicative devices / Switching devices
                                          • Communicative links

                                        • Software:

                                          • for resource sharing

                                          • tools for users

                                            e.g. email, ftp, chat, ...

                                        • Protocol: the core of computer network,

                                      • Divided by working

                                        • Edge: hosts

                                        • Core: networks and routers

                                          Provide connectivity and switch services for edge parts

                                      • Divided by function

                                        • Communication subnet: data communication, transmission media, communication devices, protocols
                                        • Resource subnet: resource sharing and data processing, devices and software

                                        The layered models are just divided by function

                                    • Classification:

                                      • Classified by distribution range

                                        • Wide area network (WAN)
                                          • Range: tens to thousands of kilometers, large, multinational
                                          • Prime technology: switch
                                          • Status: the core of internet, high speed links, large communication capacity
                                        • Metropolitan area network (MAN)
                                          • Range: five to fifty kilometers
                                          • Prime technology: ethernet
                                          • Status: always merged into LAN
                                        • Local area network (LAN)
                                          • Range: tens to thousands of meters
                                          • Prime technology: broadcast
                                        • Personal area network
                                          • Range: ten meters
                                          • Prime technology: wireless

                                      • Classified by user

                                        • Public network: large network constructed by telecom companies
                                        • Private network: constructed for business requirements of specific unit

                                      • Classified by topological structure

                                        • Bus network
                                        • Star network: nnn nodes, n‚àí1n-1n‚àí1 links
                                        • Ring network
                                        • Net network: always used in WAN like internet

                                      • Classified by switch technology

                                        • Circuit switching network
                                        • Message switching network
                                        • Packet switching network

                                        More detailed introductions later ...

                                      • Classified by transfer technology

                                        • Broadcast network: communication channel shared by all hosts

                                        • Point to point network: each link connecting a pair of computers

                                          packet store-and-forward and routing

                                    # Standardization & Layered Models

                                    • Standardization:

                                      • Why to standardize: follow the uniform standards to realize the interconnection between different hardware and software from different manufacturers, promoting the development of computer network

                                      • What category of standards:

                                        • Mandatory standards: official and legal standards set by authorities

                                          For example, OSI/ISO standard

                                        • DE facto standards: standards in some products of companies dominating the competition

                                          For example, TCP/IP standard

                                      • How to set standards:

                                        • RFC (Request For Comments)

                                          1. Internet draft
                                          2. Proposed standard: RFC document
                                          3. (Draft standard)
                                          4. Internet standard: audited by IETF and IAB

                                        • Related organizations:

                                          • ISO: International Standardization Organization

                                            OSI: Open System Interconnection

                                            HDLC: High-level Data Link Control

                                          • ITU: International Telecommunication Union

                                          • IEEE: Institute of Electrical and Electronics Engineers

                                          • IETF: the Internet Engineering Task Force

                                          • IAB: Internet Architecture Board

                                    • Layering:

                                      • What is / Why to layer: an abstract computer network architecture describing functions of network, to divide and conquer

                                      • What‚Äôs inside layers:

                                        • Entity: PDU(Protocol Data Unit) = SDU(Service Data Unit) + PCI(Protocol Control Information)

                                          • PDU‚Äôs ‚Äúnickname‚Äù in different layers:
                                            • ‚Äúbit‚Äù in physical layer
                                            • ‚Äúframe‚Äù in link layer
                                            • ‚Äúpacket‚Äù in network layer
                                            • ‚Äúmessage‚Äù in transport layer

                                        • Network protocol: horizontal rules or appointments to exchange data between peer entities

                                          • Grammar
                                          • Semantics
                                          • Sync

                                        • Interface: SAP(Service Access Point) where vertical entities exchange data, or the upper layer calls the lower layer

                                        • Service: ‚Äúvertical‚Äù function calling served by the lower layer for the upper layer

                                      • How to layer:

                                        • Principles:

                                          • Independence and complete, with appropriate technologies
                                          • Clear boundaries and interfaces
                                          • Standardization

                                        • Models: ISO/OSI, TCPIP

                                    • ISO/OSI:

                                      • Where‚Äôs ISO/OSI from: ISO raised it to support the interconnection of heterogeneous network systems
                                      • How does ISO/OSI layer:

                                        • Communication subnet:

                                          • Physical layer: transparent bit transfer on physical mediums

                                            • Interface feature
                                            • Transmission mode
                                            • Bit encoding and syncing

                                          • Data link layer: framing

                                            • Error control
                                            • Flow control
                                            • Media access control

                                          • Network layer: transmit packets from source to destination according to protocols like IP

                                            • Routing
                                            • Error control
                                            • Flow control, congestion control

                                        • Transport layer: peer-to-peer communication, between processes, with the unit of message according to protocols like TCP and UDP

                                          • Reliable or reliable transmission
                                          • Reuse and multiplexing
                                          • Error control
                                          • Flow control

                                        • Resource subnet:

                                          • Session layer: connection establishment
                                            • Checkpoint
                                          • Presentation layer: Handle information representing between two communication systems
                                            • Data format transformation
                                            • Data encryption and decryption
                                            • Data compression and decompression
                                          • Application layer: communication between application programs according to protocols like FTP, SMTP, and WWW

                                    • TCP/IP:

                                      • Where‚Äôs TCP/IP from: TCP/IP protocol stack
                                      • How does TCP/IP layer:

                                        • Network interface layer: physical and data link layer

                                          • Protocols: Ethernet, ATM, Frame Relay

                                        • Internet layer: network layer

                                          • Protocols: IP

                                        • Transmission layer: transport layer

                                          • Protocols: TCP, UDP

                                        • Application layer: session, presentation and application layer

                                          • Protocols: HTTP, FTP, DNS

                                    ISO/OSI vs TCP/IP

                                    • Similarity:
                                      • Layer and protocol stack
                                      • Interconnection of heterogeneous network
                                    • Difference:
                                      • ISO/OSI: model over protocol
                                      • TCP/IP: value IP for interconnection of heterogeneous network initially
                                      • Different communication ways in network and transport layer: connection-oriented or connectionless
                                        • connection: establish, transmit and release

                                    # Client / Server Model:

                                    • What is C/S:
                                      • Server: the device providing computing services
                                      • Client: the host requesting computing services
                                    • How does C/S work:
                                      • Server:
                                        • Provide services ‚Äúpermanently‚Äù
                                        • Permanent access address or domain name
                                      • Client:
                                        • Communicate with server and obtain services
                                        • Access the network intermittently, dynamic IP address
                                        • Do not communicate with other clients directly
                                    • What‚Äôs the use of C/S:
                                      • WEB
                                      • FTP
                                      • Remote login
                                      • Email

                                    # Peer-To-Peer Model:

                                    • What‚Äôs inside P2P:
                                      • No server
                                      • Hosts both providing services and receiving services
                                    • How does P2P work:
                                      • Direct communication between terminal systems
                                      • Access the network intermittently, dynamic IP address
                                      • Extendable and strong

                                    # Domain Name System:

                                    • Why we need DNS:

                                      • Remember domain name but not IP
                                      • Transform from domain name to IP

                                    • What‚Äôs domain name:

                                      • Components: mark number and point
                                      • Levels:
                                        • Root
                                        • Top level name: countries, universal, reverse
                                        • Second-level name: category, administrative area, user-registered
                                        • Third-level name (and more): automatically obtained

                                    • What‚Äôs domain name server:

                                      • Local domain name server: cache transforming domain names into IPs, receive DNS query of host firstly

                                        • Merits of cache: less payload, request, increase speed
                                        • Update of cache: timer

                                        Host itself also has a cache

                                      • Three-level architecture:

                                        1. Root domain name server: contain top domain names and IPs of their corresponding server

                                          There are 13 root server domain names in total globally, a.rootservers.net ... m.rootservers.net

                                        2. Top domain name server: contain second-level domain names belonging to this top and IPs of their corresponding server

                                        3. Authority domain name server: contain second-level (and more) domain names of a region and their IPs

                                    • How does DNS work:

                                      1. Local cache

                                      2. (If cache miss) Local to root

                                      3. If iteration

                                        1. Root to local
                                        2. Local to top to local
                                        3. Local to authority to local

                                      4. If recursion

                                        1. Root to top
                                        2. Top to authority
                                        3. Authority to top
                                        4. Top to root
                                        5. root to lcoal
