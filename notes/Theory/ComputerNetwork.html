                                                                    # Concept & Classification

                                                                    # Concept:

                                                                    • Concise concept: a set of interconnected and autonomous computers

                                                                      • Interconnected: communication link
                                                                      • Autonomous: no master-slave relation

                                                                    • Complete concept: a system connecting distributed and independent computer systems by communicative devices and links, and implement resource sharing and information transfer by software

                                                                      • Computer systems / Terminal systems: mobile, PC, ...
                                                                      • Communicative devices: switch, router, ...
                                                                      • Communicative links: mediums connecting terminal systems and communicative devices
                                                                      • Topological structure & Software: The topological formed by terminal systems, communicative devices and links can implement functions only by software

                                                                    Questions

                                                                    • Whatâ€™s UDP?

                                                                    • How does UDP datagram work?

                                                                    • Whatâ€™s UDP: User Datagram Protocol

                                                                      • Only implement multiplexing, demultiplexing and error detection based on IP datagram

                                                                      • Connectionless and best-effort delivery, unreliable, no ACK

                                                                        Reliable delivery can only be implemented by application layer

                                                                      • Message-oriented. Encapsulate the complete application layer message only by adding a UDP header, no division.

                                                                        So the length of application layer message is vital

                                                                      • No congestion control

                                                                        Solutions when serious congestion: FEC(Forward Error Correct), retransmit, ...

                                                                      • Low header cost, only 8 Bytes

                                                                        TCP 20 Bytes

                                                                    • Why we use UDP:

                                                                      • Low delay for connectionless
                                                                      • Small file for message-oriented
                                                                      • Real-time communication for no congestion control

                                                                    • How does UDP datagram work:

                                                                      • Format of header:

                                                                        • Source port (2 Bytes): Not essential, depending on response
                                                                        • Destination port (2 Bytes): Essential. If port missing when demultiplexing, discard datagram and send ICMP â€œport unreachableâ€ error report message to sender
                                                                        • UDP length (2 Bytes): The length of the whole UDP datagram. The unit is byte.
                                                                        • UDP checksum (2 Bytes): Check if the whole UDP datagram has error. If error, discard
                                                                        • (Data: Message-Oriented, no division)

                                                                      • UDP checksum:

                                                                        • Pseudo-header: Only occur when checksum, in front of UDP header

                                                                          • Source IP address (2 Bytes)
                                                                          • Destination IP address (2 Bytes)
                                                                          • All zero filed (1 Bytes)
                                                                          • Protocol field (1 Bytes): Always 17, represent UDP datagram is encapsulated in IP datagram
                                                                          • Length of UDP (2 Bytes): Exclude pseudo-header

                                                                          Pseudo IP header

                                                                        • Checksum process:

                                                                          • Sender:
                                                                            1. Fill in pseudo-header
                                                                            2. Fill in checksum field with all zeros
                                                                            3. Fill in data part with zero to integral multiple of 4 Bytes
                                                                            4. Do 16-bit binary inverse code summation with pseudo-header, header and data
                                                                            5. Fill in checksum field with summation result
                                                                            6. Remove pseudo-header and send
                                                                          • Receiver:
                                                                            1. Fill in pseudo-header
                                                                            2. Do 16-bit binary inverse code summation with pseudo-header, header and data
                                                                            3. If the result is all ones, no error, else discard datagram or deliver to application layer with error warning
